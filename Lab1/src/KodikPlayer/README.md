# 🎬 Kodik Player Integration Guide

## Описание

Встроенный компонент React для просмотра видео из базы Kodik с поддержкой выбора серии, озвучки и совместного просмотра.

---

## ✨ Новые возможности (v2.1)

### 🎯 Выбор серии и озвучки

Компонент предоставляет интерфейс для выбора предпочтительной серии и озвучки аниме:

- **📺 Выбор серии**: Dropdown со списком всех доступных серий
- **🔊 Выбор озвучки**: Dropdown с доступными русскими озвучками
- **🔗 Внешняя ссылка**: Открытие плеера на сайте Kodik для просмотра

### 🎨 Улучшенный UI

- Анимированные dropdown'ы с glassmorphism эффектами
- Инструкции по использованию Kodik
- Responsive дизайн для мобильных устройств
- Graceful fallback при ошибках загрузки

---

## 🎯 Функциональность выбора серии/озвучки

### Как это работает:

1. **Загрузка метаданных**: При получении `shikimoriId` компонент:
   - Получает количество серий из Shikimori
   - Создает список серий (1, 2, 3, ..., N)
   - Создает список русских озвучек

2. **Выбор предпочтений**: Пользователь выбирает желаемую серию и озвучку в интерфейсе
3. **Открытие на Kodik**: Нажимает кнопку "Открыть на Kodik"
4. **Просмотр на сайте**: Выбор серии и озвучки происходит непосредственно на сайте Kodik

### Доступные озвучки (только русские):
- **AniLibria** - Популярная качественная озвучка
- **AniDUB** - Качественная озвучка
- **2x2** - Озвучка канала 2x2
- **SHIZA Project** - Профессиональная озвучка

### Пример использования с выбором серии:

```jsx
<KodikPlayer 
  title="Наруто"
  shikimoriId="z20"
  allowWatchTogether={true}
  roomId="room-naruto"
/>
```

После загрузки появятся два dropdown'а:
- **Серия**: 1, 2, 3, ..., 220
- **Озвучка**: AniLibria, AniDUB, 2x2, SHIZA Project

## 🔧 Техническая реализация

### Источники данных (только русские источники)

Компонент использует следующие источники данных:

1. **Shikimori.one** - для получения основной информации об аниме (количество серий, описание)
2. **Kodik.info** - для поиска и просмотра видео через веб-интерфейс

### Архитектура

```
KodikPlayer
├── getData()           // Получение данных о сериях и озвучках
├── updatePlayerUrl()   // Обновление состояния выбора (не URL)
├── openExternal()     // Открытие поиска Kodik в новой вкладке
└── handleSelection()  // Обработка выбора серии/озвучки
```

### Работа с Kodik

Компонент открывает поиск Kodik по Shikimori ID:
```
https://kodik.info/search?types=anime&shikimori_id=z20
```

**Примечание**: Прямое указание серии и озвучки в URL не поддерживается Kodik, поэтому выбор происходит непосредственно на сайте Kodik после открытия ссылки.

## 📦 Компоненты

### KodikPlayer

Основной компонент для встраивания плеера Kodik.

#### Props

```typescript
interface KodikPlayerProps {
  videoUrl?: string;           // Прямой URL видео
  shikimoriId?: string;        // ID из Shikimori (например: z20 для Naruto)
  kodikId?: string;            // ID из Kodik
  title?: string;              // Название видео
  allowWatchTogether?: boolean; // Включить совместный просмотр
  roomId?: string;             // ID комнаты для совместного просмотра
}
```

#### Примеры использования

**Базовый пример:**
```jsx
import KodikPlayer from './KodikPlayer/KodikPlayer';

<KodikPlayer 
  title="Naruto"
  shikimoriId="z20"
/>
```

**С совместным просмотром:**
```jsx
<KodikPlayer 
  title="One Piece"
  shikimoriId="5"
  allowWatchTogether={true}
  roomId="room-5"
/>
```

**С URL:**
```jsx
<KodikPlayer 
  title="My Video"
  videoUrl="https://kodik.info/video/..."
/>
```

## 🔍 Как найти ID

### Shikimori ID

1. Зайдите на [Shikimori.one](https://shikimori.one)
2. Найдите нужное аниме
3. Посмотрите ID в адресной строке:
   - `https://shikimori.one/animes/z20-naruto` → ID: `z20`
   - `https://shikimori.one/animes/5-cowboy-bebop` → ID: `5`

### Kodik ID

1. Откройте плеер Kodik
2. Найдите видео
3. ID будет в URL видео

## 🌐 Совместный просмотр

### Как это работает

1. Нажмите кнопку "🎬 Смотреть вместе"
2. Скопируется ссылка на комнату
3. Поделитесь со своим другом
4. Когда друг откроет ссылку, вы сможете смотреть синхронно

### Синхронизация

- **WebSocket**: Real-time синхронизация действий
- **Flask-SocketIO**: Бэкенд для обработки подключений
- **Задержка**: < 1 секунды между пользователями
